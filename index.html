API Description (JIRA Ticket #AUT-101)

Project Context

What API: This is a RESTful Authentication API (Login API) that handles user credential validation and session token generation.

Why it will be used: It serves as a controlled sandbox for API Automation Testing. It will be used to practice schema validation, negative testing, and security assertions using tools like Playwright and Supertest.

How it should work:
  1. Request: Client sends a POST request with email and password.
  2. Format Check: System validates email format and ensures password is ≥ 8 chars (returns 422 if fail).
  3. Verification: System verifies credentials against the database (returns 401 or 404 if fail).
  4. Security: After 5 consecutive failures, the account is locked for 30 minutes (returns 423).
  5. Success: Returns 200 OK with a 24-hour JWT token and the user profile object.

The following is the plain-text API specification from the development ticket. Use this as a reference for expected request and response structures. 

Endpoint: POST /auth/login

### Request Body (application/json):
{
  "email": "user@example.com",      // required, valid email format
  "password": "Secret@123"          // required, minimum 8 characters
}

### Responses:

200 — Login Successful:
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "tokenType": "Bearer",
  "expiresIn": 86400,
  "user": {
    "id": "usr_01HXYZ123456",
    "name": "John Doe",
    "email": "user@example.com",
    "role": "admin"         // enum: admin, editor, viewer
  }
}

400 — Bad Request (missing/empty fields):
{
  "status": 400,
  "error": "Bad Request",
  "message": "Email and password are required"
}

401 — Unauthorized (wrong password):
{
  "status": 401,
  "error": "Unauthorized",
  "message": "Invalid email or password"
}

404 — Not Found (email doesn't exist):
{
  "status": 404,
  "error": "Not Found",
  "message": "No account found with this email address"
}

422 — Validation Error (invalid format):
{
  "status": 422,
  "error": "Unprocessable Entity",
  "message": "Validation failed",
  "details": [
    { "field": "email", "issue": "Must be a valid email address" },
    { "field": "password", "issue": "Minimum 8 characters required" }
  ]
}

423 — Locked (5 failed attempts):
{
  "status": 423,
  "error": "Locked",
  "message": "Account locked due to too many failed attempts. Try again after 30 minutes."
}

500 — Internal Server Error:
{
  "status": 500,
  "error": "Internal Server Error",
  "message": "Something went wrong. Please try again later."
}
